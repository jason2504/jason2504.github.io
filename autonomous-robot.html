Project Overview

This MEAM 5100 Design of Mechatronic Systems project challenged teams to design, build, and program autonomous mobile robots for a League of Legends-inspired battle arena competition. Our robot successfully demonstrated both autonomous navigation capabilities and manual control through a custom web interface, achieving perfect performance across all evaluation criteria in a highly competitive environment.

The RoBA (Robot Battle Arena) competition featured head-to-head matches where teams attacked opposing bases (Nexus), captured neutral towers, and defended their own assets. Each robot carried a "Top Hat" module that tracked health points, enforced game rules, and penalized excessive WiFi communication, requiring careful balance between autonomous operation and manual control strategies.

Game Rules and Strategic Constraints

The competition environment imposed several critical constraints that shaped our design approach:

  • Health System: Robots started with fixed HP, losing 10 HP per physical hit with 0.5s cooldown
  • Communication Cost: Each WiFi packet cost 1 HP, penalizing overuse of remote communication
  • Arena Features: Central ramp, multiple towers, and Nexus targets requiring precise navigation
  • Victory Conditions: Direct Nexus destruction or gradual damage through captured tower control

Competition arena featuring central ramp, Nexus targets, and tower positions

Technical Innovation: Dual-MCU Architecture

Our design employed two cooperating ESP32-S2 microcontrollers with specialized responsibilities to ensure robust real-time performance. The Sensor MCU handled high-level planning and sensor fusion, while the Motor MCU focused exclusively on low-level control with encoder feedback. This separation ensured time-critical motor actuation remained unaffected by higher-level computational workloads.

Complete system wiring diagram showing ESP32, sensors, motors, and Vive detection circuits

Advanced Control Systems

The robot featured sophisticated autonomous navigation using PD control for wall following, combined with Vive localization for precise positioning during attack sequences. Manual control was implemented through a custom web interface hosted directly on the ESP32, providing real-time WASD keyboard controls without requiring additional software installation.

System Architecture and Technical Design

Mechanical Design Philosophy

The robot's 3-layer acrylic chassis design prioritized modularity, accessibility, and protection:

Structural Components
  • Laser-cut acrylic frame with 3D-printed motor mounts
  • Parallax Feedback 360¬∞ servo motors for precise control
  • Custom drive wheels with rubber band traction enhancement
  • MG995 servo-driven attack arm with 90¬∞ sweep capability
Protection and Accessibility
  • Protective bumper system for robot-to-robot contact
  • Modular sensor housing for easy maintenance access
  • Cable management system preventing interference
  • Caster wheel design for stable omnidirectional support

Control Systems Implementation

Autonomous Control
  • PD controller for wall following with tuned parameters
  • Vive sensor integration for precise localization
  • Automated attack sequences at calibrated coordinates
  • Robust sensor fusion for navigation decisions
Manual Control Interface
  • Real-time web interface hosted on ESP32
  • WASD keyboard controls with visual feedback
  • Closed-loop/open-loop mode switching
  • Health monitoring and game state display

Electronic Systems and Communication

The electronic architecture emphasized reliability and real-time performance:

  • Dual-MCU Design: ESP32-S2 controllers with specialized task distribution for optimal performance
  • Sensor Integration: HC-SR04 ultrasonic sensors for wall detection and obstacle avoidance
  • Power Management: Efficient power distribution with dedicated motor drivers and control circuits
  • Communication Protocol: I2C master-slave architecture ensuring deterministic control timing

Technical Challenges and Solutions

Real-Time Control Coordination

Challenge: Ensuring motor control remained responsive while processing sensor data and high-level navigation decisions.

Solution: Implemented dual-MCU architecture with dedicated motor controller, preventing sensor processing delays from affecting critical actuator timing.

Sensor Reliability in Competition Environment

Challenge: Maintaining consistent ultrasonic sensor performance despite interference from multiple robots and arena acoustics.

Solution: Developed robust sensor fusion algorithms with outlier rejection and implemented backup navigation strategies for sensor failure scenarios.

Power and Communication Management

Challenge: Balancing WiFi communication needs with HP conservation requirements (1 HP per packet).

Solution: Optimized communication protocols with efficient data packaging and implemented smart mode switching to minimize unnecessary transmissions.

Mechanical Durability Under Competition Stress

Challenge: Ensuring robot survived physical contact during competitive matches while maintaining precision.

Solution: Designed protective bumper systems and robust mounting strategies, with easily replaceable components for rapid field repairs.
