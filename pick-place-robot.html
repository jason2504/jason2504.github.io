Pick-and-Place Robot

Vision-Guided Manipulation with 7-DOF Franka Panda

Successfully implemented AprilTag detection, inverse kinematics optimization, and time-based motion prediction for both static and dynamic block manipulation, demonstrating 100% simulation success rates and robust real-world performance with 7-DOF Franka Panda manipulator.

Project Overview

This MEAM 5200 Introduction to Robotics final project challenged teams to develop autonomous pick-and-place capabilities using the sophisticated Franka Emika Panda 7-DOF manipulator. Our interdisciplinary team "Robust Robotics" successfully integrated computer vision, inverse kinematics, trajectory planning, and dynamic prediction to achieve 4th place in a highly competitive field, demonstrating both technical excellence and strategic innovation.

The project required robots to autonomously detect, grasp, and stack wooden blocks from both static platforms and a rotating turntable under time constraints. This challenge demanded seamless integration of perception, planning, and control systems while adapting to both stationary and moving objects in a dynamic competitive environment.

Competition Environment and Setup

The competition arena featured multiple platforms with distinct scoring systems designed to test different aspects of robotic manipulation. Static blocks on fixed platforms were worth 10 points per millimeter of stacking height, while dynamic blocks on the rotating turntable offered 20 points per millimeter, creating strategic risk-reward decisions that shaped our approach to maximizing competitive performance. Each block is 50mm in height and the stacking height is counted at the centroid of each stacked block.

Competition arena showing Franka Panda setup with static platforms, rotating turntable, and goal areas

System Architecture and Control Framework

Our solution employed a modular architecture organized around two primary components: the ArmController for kinematic planning and trajectory execution, and the ObjectDetector for vision-based perception and pose estimation. This separation enabled continuous block localization while maintaining stable motion planning, creating a robust pipeline for both static and dynamic manipulation tasks.

Strategic Innovation: Two-Phase Stacking Approach

Our team developed a sophisticated stacking strategy to maximize points while ensuring reliability:

  • Phase 1: Create separate 4-block stacks of dynamic and static blocks to establish stable foundations
  • Phase 2: Combine stacks by carefully moving dynamic blocks on top of static stack for maximum height
  • Theoretical Maximum: 8-block tower achieving 28,000 points under optimal conditions
  • Reliability Features: Camera verification after each placement and failsafe recovery protocols

Technical Implementation and Results

Static Block Manipulation Pipeline

Simulation: 4-block static stacking

Real-world: 4-block static stacking

The static block pipeline achieved highly reliable performance through precise AprilTag detection and camera calibration. Once detected, block poses were computed in the world frame using homogeneous transformations, enabling the ArmController to generate accurate alignment trajectories.

Dynamic Block Manipulation with Prediction

Simulation: 4-block dynamic stacking

Real-world: 4-block dynamic stacking

Dynamic block manipulation required sophisticated prediction algorithms accounting for turntable rotation at 0.4 RPM. Our system estimated angular velocity, calculated end-effector travel time, and extrapolated future block positions using rotation matrices for successful interception trajectories.

Advanced Stacking Strategy

Simulation: 4 dynamic blocks stacked on 4 static blocks for maximum points

Our two-phase approach maximized scoring potential by first establishing stable foundations, then carefully combining stacks. The inverse kinematics solver maintained alignment across multiple placements, while incremental z-offsets ensured precise placement on the narrow scoring platform.

  • Phase 1: Separate 4-block stacks (foundation)
  • Phase 2: Combine for 8-block tower (28,000 points)
  • Safety features: Camera verification and failure recovery

Vision System and Control Architecture

Computer Vision Pipeline
  • AprilTag detection with Intel RealSense D435 camera
  • Camera calibration correction for accurate pose estimation
  • Homogeneous transformation from camera to world frame
  • Optimal detection heights: 0.5m static, 0.45m dynamic platforms
Motion Control and Kinematics
  • IK_position_null with joint centering secondary task
  • Gradient descent optimization with pseudo-inverse
  • 0.001 linear and angular tolerance for precision
  • Optimized arm speed at 0.2 for stability and accuracy
